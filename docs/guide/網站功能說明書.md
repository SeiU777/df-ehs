## 第一章：網站總覽

### 1.1 網站是什麼？

大豐環保法務及環安課的**職業災害分析報告網站**，部署在 GitHub Pages 上，所有員工可透過網址瀏覽。

**網址**：https://seiu777.github.io/df-ehs/

### 1.2 網站包含的內容

| 區塊 | 說明 | 頁面數 |
|------|------|--------|
| **首頁** | KPI 總覽 + 報告入口卡片 | 1 頁 |
| **年度報告** | 2023 / 2024 / 2025 各年度職災分析 | 6 頁（各年完整版+摘要版） |
| **三年總覽** | 2023-2025 跨年趨勢對比 | 2 頁（完整版+摘要版） |
| **環境因素分析** | 天氣、時間、地理等 6 大維度 | 2 頁（完整版+摘要版） |
| **安委會季報** | 法定 12 項辦理事項季度報告 | 3 頁（首頁+上傳工具+Q1 報告） |
| **教育訓練** | 安全衛生教育課程（含測驗） | 2 頁（總公司版+全興廠版） |

### 1.3 技術特色

- **純靜態網站**：HTML + CSS + JavaScript，無後端伺服器
- **密碼保護**：XOR 加密驗證，sessionStorage 保持登入
- **深色模式**：支援淺色/深色主題切換，跟隨系統設定
- **響應式設計**：手機、平板、電腦三種螢幕尺寸
- **Chart.js 圖表**：互動式統計圖表（折線圖、圓餅圖、長條圖）
- **側邊目錄**：可收合的章節導航（桌面常駐、行動裝置隱藏）

## 第二章：密碼保護系統

### 2.1 密碼

```
Da0302fon2026%
```

### 2.2 登入方式

1. 開啟網站首頁，畫面會顯示全螢幕的密碼輸入框
2. 輸入密碼 `Da0302fon2026%`
3. 按下登入按鈕或 Enter 鍵
4. 驗證成功後，密碼層消失，顯示網站內容

### 2.3 登入狀態

- 登入後，瀏覽同一視窗中的所有頁面**不需要再次輸入密碼**
- 狀態儲存在 `sessionStorage`（關鍵：`df-ehs-auth`）
- **關閉瀏覽器**或**關閉分頁**後會自動登出，下次需重新輸入

### 2.4 哪些頁面有密碼保護？

| 頁面 | 密碼保護 |
|------|:--------:|
| 首頁 index.html | 有 |
| 年度報告（2023/2024/2025） | 有 |
| 年度報告摘要版 | 有 |
| 三年總覽 | 有 |
| 環境因素分析 | 有 |
| 安委會季報 | 有 |
| 教育訓練 | 有 |

### 2.5 技術細節

密碼驗證邏輯在 `js/auth.js` 中：
- 加密方式：XOR（金鑰 = 42）
- 編碼後資料：`[110,75,26,25,26,24,76,69,68,24,26,24,28,15]`
- 驗證函式：逐字元比對 `input.charCodeAt(i) ^ 42 === data[i]`
- **非高安全加密**，僅防止一般瀏覽，有技術背景的人可解碼

### 2.6 如何修改密碼

1. 開啟 `js/auth.js`
2. 決定新密碼（例如 `NewPass2026!`）
3. 用以下 JavaScript 計算新的編碼資料：
```javascript
const password = 'NewPass2026!';
const key = 42;
const encoded = Array.from(password).map(c => c.charCodeAt(0) ^ key);
console.log(encoded); // 貼入 auth.js 的 _d 陣列
```
4. 將結果替換 `_d` 陣列
5. 提交並推送

## 第四章：年度職災報告

### 4.1 共有三個年度

| 年度 | 檔案 | 主題色 |
|------|------|--------|
| 2023 | `reports/2023.html` | 綠色漸層 |
| 2024 | `reports/2024.html` | 橙色漸層 |
| 2025 | `reports/2025.html` | 紅色漸層 |

### 4.2 報告結構（每年相同）

每份年度報告包含以下章節：

1. **封面** — 年度標題、資料期間、公司名稱
2. **KPI 總覽** — 4 個指標卡片（總案件數 / 損失工日 / 嚴重事故 / 月均件數）
3. **事故分類統計** — 工安 / 交通 / 虛驚的件數與佔比（圓餅圖）
4. **事故類型深度分析** — 各類型（被夾被捲、跌倒、墜落等）的件數與嚴重度
5. **月份趨勢** — 每月事故件數折線圖 / 長條圖
6. **部門分析** — 各部門事故頻率
7. **原因分析** — 不安全行為 / 不安全狀況 / 基本原因統計
8. **嚴重事故案例** — 重大案件詳細分析
9. **改善建議** — 工程 / 管理 / 教育 / PPE 四層次
10. **結語** — 年度總結與展望

### 4.3 使用的視覺元件

| 元件 | CSS Class | 用途 |
|------|-----------|------|
| KPI 卡片 | `.kpi-grid` + `.kpi-card` | 指標數字展示 |
| 圓餅圖 | CSS `conic-gradient` | 比例分佈圖（純 CSS，無外部套件） |
| 長條圖 | CSS `width` 百分比 | 頻率 / 嚴重度比較 |
| Chart.js 圖表 | `<canvas>` | 互動式折線圖、長條圖 |
| 重點卡片 | `.highlight-card` | 綠色邊框，正面指標 |
| 警示卡片 | `.warning-card` | 橙色邊框，需注意項目 |
| 危險卡片 | `.danger-card` | 紅色邊框，嚴重警示 |
| 徽章 | `.badge-danger/warning/success` | 嚴重度標籤 |
| 側邊目錄 | `.sidebar` + `.toc-chapter` | 章節導航（可收合） |

### 4.4 側邊目錄功能

- **桌面（>1024px）**：左側常駐顯示，點擊章節跳轉
- **平板/手機（≤1024px）**：預設隱藏，點擊選單按鈕開啟
- 點擊章節連結後自動關閉（行動裝置）
- 半透明遮罩層覆蓋背景

## 第九章：深色/淺色模式

### 9.1 切換方式

- 頁面右下角有一個圓形按鈕
- 淺色模式顯示 🌙（點擊切換到深色）
- 深色模式顯示 ☀️（點擊切換到淺色）

### 9.2 運作邏輯

1. **首次訪問**：跟隨系統設定（Windows 深色模式 → 網站深色模式）
2. **手動切換**：偏好儲存在 `localStorage`（關鍵：`theme`）
3. **持久記憶**：關閉瀏覽器後再開，仍保持上次選擇
4. **系統跟隨**：如果使用者未手動切換，會自動跟隨系統主題變更

### 9.3 技術實作

- `js/theme.js` — 切換邏輯
- `css/dark.css` — 深色模式的 CSS 變數覆蓋
- HTML 屬性：`<html data-theme="dark">` 或 `<html data-theme="light">`

## 第十三章：JavaScript 模組

### 13.1 auth.js — 密碼驗證

**公開 API**（`window.dfAuth`）：

| 方法 | 說明 |
|------|------|
| `dfAuth.verify(input)` | 驗證輸入的密碼是否正確，回傳 boolean |
| `dfAuth.isAuthenticated()` | 檢查是否已登入（sessionStorage） |
| `dfAuth.setAuthenticated()` | 設定已登入狀態 |
| `dfAuth.logout()` | 登出（清除 sessionStorage） |

- 頁面載入時讀取 `localStorage.theme` 或系統偏好

### 13.3 sidebar.js — 側邊目錄

| 函式 | 說明 |

## 第十四章：如何修改與更新

### 14.1 更新報告數據

1. 取得最新職災 Excel
2. 用 Claude Code 說「資料更新了，幫我更新報告」
3. Claude 讀取 Excel → 更新 HTML 中的數據、圖表、KPI
4. `git add` → `git commit` → `git push`

### 14.2 新增年度報告

1. 複製上一年的 HTML（如 `2025.html` → `2026.html`）
2. 修改：標題、封面顏色（每年不同色系）、sidebar 目錄
3. 替換所有數據和圖表
4. 在 `index.html` 新增年度卡片
5. 在 `overview.html` 加入新年度數據
6. 建立對應的摘要版 `2026-summary.html`

### 14.3 新增安委會季報

1. 複製 `committee/2026-Q1.html` → `committee/2026-Q2.html`
2. 修改內容（法定 12 項各項更新）
3. 在 `committee/index.html` 啟用 Q2 卡片（移除 disabled）
4. 將素材照片放到 `committee/images/2026-Q2/`

### 14.4 修改教育訓練

- 編輯 `training/hq.html` 或 `training/quanxing.html`
- 測驗題的正確答案在 `.quiz-question` 的 `data-answer` 屬性中

### 14.5 修改密碼

見 [第二章 2.6 節](#26-如何修改密碼)

### 14.6 修改樣式

- 改品牌色 / 字型 → 編輯 `css/common.css` 的 `:root` 變數
- 改深色模式 → 編輯 `css/dark.css`
- 改側邊欄 → 編輯 `css/sidebar.css`
- 改教育訓練外觀 → 編輯 `css/training.css`
