## 第一章：專案簡介與系統架構

### 1.1 這個系統是什麼？

這是大豐環保法務及環安課的**職業災害分析報告系統**，功能包括：

- 職災統計報告網站（GitHub Pages 自動部署）
- 個案職災分析報告產生器（AI 輔助）
- 安委會季度簡報生成工具
- 教育訓練教材網站

### 1.2 系統運作流程

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  Excel 職災資料  │────→│   Claude Code    │────→│  HTML 報告檔案   │
│ （每月更新）      │     │ （AI 讀取分析）    │     │ （df-ehs/docs/） │
└─────────────────┘     └──────────────────┘     └────────┬────────┘
                                                          │ git push
                                                          ▼
                                                 ┌─────────────────┐
                                                 │  GitHub Pages   │
                                                 │ （自動部署網站）  │
                                                 └─────────────────┘
                                                          │
                                                          ▼
                                          https://seiu777.github.io/df-ehs/
```

### 1.3 四大組件

| 組件 | 技術 | 用途 |
|------|------|------|
| **A. 靜態網站** | HTML + CSS + JS | 職災報告網站，部署在 GitHub Pages |
| **B. Python 腳本** | Python 3.12+ | 讀取 Excel、處理 PDF、資料分析 |
| **C. 簡報工具** | Node.js | 生成安委會 PowerPoint/Word |
| **D. Claude Code** | AI 助手 | 自動更新報告、分析職災案件 |

## 第三章：環境安裝

### 3.1 快速安裝（推薦）

如果你的電腦已經有 Git、Python、Node.js：

1. 把整個 `職安_完整專案包/` 複製到 `C:\Users\{你的使用者名稱}\Documents\claude\職安\`
2. 雙擊執行 `setup.bat`
3. 完成！

### 3.2 從零開始安裝

#### 第一步：安裝 Git

1. 前往 https://git-scm.com/download/win 下載
2. 安裝時保持預設選項
3. 驗證：開啟命令提示字元，輸入 `git --version`

#### 第二步：安裝 Python

1. 前往 https://www.python.org/downloads/ 下載 3.12 以上版本
2. **安裝時務必勾選 "Add Python to PATH"**
3. 驗證：`python --version`

> 如果顯示「不是內部或外部命令」，重新安裝並勾選 "Add Python to PATH"

#### 第三步：安裝 Node.js

1. 前往 https://nodejs.org/ 下載 LTS 版本
2. 安裝時保持預設選項
3. 驗證：`node --version`

#### 第四步：安裝 VS Code

1. 前往 https://code.visualstudio.com/ 下載
2. 建議擴充功能：**Claude Code**（Anthropic 官方）

#### 第五步：放置專案

將整個資料夾複製到建議路徑：

```
C:\Users\{你的使用者名稱}\Documents\claude\職安\
```

> 建議使用與原開發者相同的路徑，可減少設定修改。

#### 第六步：執行安裝腳本

雙擊 `setup.bat`，它會自動：
- 檢查 Git、Python、Node.js 是否已安裝
- 安裝 Python 依賴（openpyxl、pdfplumber、pandas 等）
- 安裝 Node.js 依賴（pptxgenjs、docx）
- 複製 Claude Code 設定和 Skills

#### 第七步：測試網站

```bash
cd df-ehs\docs
python -m http.server 3300
```

開啟瀏覽器輸入 http://localhost:3300 ，看到職災報告首頁即成功。

## 第四章：參考資料設定

### 4.1 為什麼參考資料這麼大？

參考資料包含 PDF 手冊、圖片、Excel 原始數據等，總計超過 1.1GB。
因為含有員工個資，不適合放在 GitHub 上。

### 4.2 核心檔案（必須有的）

| 檔案 | 用途 |
|------|------|
| `參考資料/公司職災資料/職災統計__2023.1-2025.12.xlsx` | 所有報告的數據來源 |
| `參考資料/公司職災資料/公司表單之事故選單內容.docx` | 事故下拉選單定義 |
| `參考資料/安委會/pptx-workspace/package.json` | 簡報工具依賴 |
| `參考資料/職災分析方法/職災分析方法手冊/` | 事故調查指南手冊 |
| `參考資料/職災分析方法/人為失效模式/HSG48_中文版.html` | 人因分析框架 |

### 4.3 資訊安全提醒

- 職災 Excel 包含**員工個人資料**（姓名、身分證字號等）
- **嚴禁上傳到 GitHub 或公開平台**
- **嚴禁透過未加密管道傳送**
- 離開電腦時請鎖定螢幕

## 第五章：Claude Code 設定

### 5.1 什麼是 Claude Code？

Claude Code 是 Anthropic 出品的 AI 編程助手，能直接操作你電腦上的檔案。
在這個專案中，它負責：
- 讀取 Excel → 自動更新 HTML 報告
- 分析事故截圖 → 產出 .docx 分析報告
- 輔助安委會簡報製作

**不是必要工具**，沒有它也可以手動編輯報告。

### 5.2 安裝方式

在 VS Code 中搜尋擴充功能 "Claude Code"（Anthropic 官方），安裝即可。
需要 Claude Max / Claude Team / Claude Enterprise 訂閱。

### 5.3 設定檔案

`setup.bat` 已自動複製以下設定：

| 檔案 | 位置 | 功能 |
|------|------|------|
| `CLAUDE.md` | `~/.claude/` | 全域指令（繁中回覆、程式碼風格） |
| `settings.json` | `~/.claude/` | 全域設定（Shell 路徑） |

### 5.4 修改 Shell 路徑（重要！）

開啟 `~/.claude/settings.json`，確認 SHELL 路徑指向你電腦的 Git Bash：

```json
{
  "env": {
    "SHELL": "C:\\Users\\{你的使用者名稱}\\AppData\\Local\\Programs\\Git\\bin\\bash.exe"
  }
}
```

> 路徑可能在 `C:\Program Files\Git\bin\bash.exe`，視安裝位置而定。

### 5.5 Skills 說明

`setup.bat` 已自動複製 3 個職安專用 Skills 到 `~/.claude/skills/`：

| Skill | 觸發方式 | 輸出 |
|-------|---------|------|
| `7-accident-analysis` | 「這是公司新的職災案件，請你幫我做分析報告」 | .docx |
| `7-occupational-safety-analyzer` | 「幫我分析公司職災資料」 | .html |
| `dafong-html-report` | 需要大豐風格報告時自動套用 | .html |

### 5.6 修改 Skills 中的硬編碼路徑（重要！）

Skills 檔案中有寫死的路徑，如果你的專案路徑與原開發者不同，需要修改：

**檔案**：`~/.claude/skills/7-accident-analysis/SKILL.md`

找到類似這樣的路徑：
```
{homedir}/Documents/claude/職安/參考資料/...
```

**檔案**：`~/.claude/skills/7-accident-analysis/report-template.js`（第 9 行）

如果你的專案放在 `Documents/claude/職安/` 路徑下，則不需要修改。

## 第六章：日常操作流程

### 6.1 月度報告更新

這是最常用的流程：每月更新職災數據後重新生成報告。

**步驟**：

1. 取得最新的職災 Excel 檔案，放到 `參考資料/公司職災資料/` 目錄
2. 開啟 VS Code，確認在職安工作區
3. 開啟 Claude Code（VS Code 側邊面板或終端機）
4. 輸入：「資料更新了，幫我更新報告」
5. Claude 會自動讀取 Excel → 更新所有 HTML 報告
6. 完成後，在終端機執行：

```bash
cd df-ehs
git add -A
git commit -m "更新 YYYY-MM 月報"
git push
```

7. 等待 1-2 分鐘，GitHub Pages 自動更新
8. 前往 https://seiu777.github.io/df-ehs/ 確認

### 6.2 個案職災分析

當發生新的職災案件，需要產出分析報告時：

**步驟**：

1. 準備事故表單截圖（發生單位表單 + 工安單位表單）
2. 開啟 Claude Code
3. 輸入：「這是公司新的職災案件，請你幫我做分析報告」
4. 提供截圖（貼上或給檔案路徑）
5. Claude 會：
   - 辨識截圖中所有欄位
   - 讀取公司下拉選單定義
   - 套用 HSG48 人為失效分析框架
   - 執行 5-Why 因果分析
   - 產出 .docx 分析報告
6. 報告存放在 `參考資料/公司職災資料/分析報告/{年份}/`

### 6.3 安委會簡報

每季安委會需要製作簡報時：

**步驟**：

1. 準備素材（照片、數據、會議紀錄）放到 `參考資料/安委會/安委會{年份}_Q{季度}/`
2. 使用 Claude Code 或 pptx-workspace 工具製作簡報
3. 安委會季報 HTML 存放在 `df-ehs/docs/reports/committee/`

### 6.4 本地預覽網站

```bash
cd df-ehs\docs
python -m http.server 3300
```

瀏覽器開啟 http://localhost:3300

結束預覽：按 `Ctrl+C`

## 第七章：常見問題排解

### Q: `python` 指令找不到？

Windows 上有時需要用 `py` 代替：

```
py --version
py -m http.server 3300
```

或重新安裝 Python 並勾選 "Add Python to PATH"。

### Q: `git push` 被拒絕？

你需要 `seiu777/df-ehs` repo 的存取權限。
請聯繫 repo 管理員，將你的 GitHub 帳號加為 collaborator。

### Q: 網站更新後看不到變化？

1. GitHub Pages 有 1-2 分鐘部署延遲
2. 清除瀏覽器快取：按 `Ctrl+Shift+R`（強制重新載入）

### Q: 中文資料夾名稱有問題？

```bash
git config --global core.quotepath false
```

### Q: npm install 失敗？

如果公司有代理伺服器：

```bash
npm config set proxy http://proxy.company.com:8080
npm config set https-proxy http://proxy.company.com:8080
```

### Q: Claude Code 找不到某個檔案？

確認 `參考資料/` 資料夾已正確放置，且路徑與 Skills 中的設定一致。

### Q: Skills 沒有出現？

確認 SKILL.md 放在 `~/.claude/skills/{skill-name}/SKILL.md`。
資料夾名稱必須與 Skill 名稱一致。

### Q: setup.bat 執行時閃退？

右鍵 setup.bat → 「以系統管理員身分執行」。
