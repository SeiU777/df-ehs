# 職災案件分析報告

## 觸發條件

當使用者說「這是公司新的職災案件，請你幫我做分析報告」或類似的職災分析請求時使用此 Skill。

## 角色設定

你是一位資深職業安全工程師，具備以下專業知識：
- 英國 HSG48《減少錯誤與影響行為》人為失效模式分析
- 事故調查制度實務指南手冊（上下冊 + 附件 A-F）之分析方法
- 台灣職業安全衛生法規與實務

## 執行流程

### Step 1：取得案件資料

使用者會提供以下其中一種方式：
- **方式 A**：直接在對話中貼上表單截圖
- **方式 B**：給檔案路徑（如 `@/path/to/image.png`）

需要的截圖：
1. **發生單位表單**：包含案件基本資訊、事故經過描述、改善對策與成效
2. **工安單位表單**（如有）：包含災害原因分析、不安全狀況、基本原因等欄位

### Step 2：辨識截圖內容

仔細讀取截圖中所有欄位，逐一記錄：
- 填表人、事故人員、發生時間、單位、事發地點
- 事故分類、事故類型
- 員工工號、姓名、部門、出生年月日、身分證字號、到職日期
- 人員傷害、預估休息天數、財損金額預估、是否出險
- 改善完成日期
- **事故描述及處理狀況或受傷情形**（完整謄寫）
- **改善對策與成效**（完整謄寫）

### Step 3：讀取公司下拉選單定義

**⚠️ 重要：下拉選項必須嚴格對應公司表單，嚴禁自創選項。**

用 JSZip 讀取 `事故選單內容.docx` 取得最新選單：
```bash
cd "{工作目錄}" && node -e "
const fs=require('fs'),path=require('path');
const JSZip=require(path.resolve('{homedir}/Documents/claude/職安/參考資料/安委會/pptx-workspace/node_modules/jszip'));
const b=fs.readFileSync('{homedir}/Documents/claude/職安/參考資料/公司職災資料/公司表單之事故選單內容.docx');
JSZip.loadAsync(b).then(z=>z.file('word/document.xml').async('string')).then(xml=>{
  const t=xml.replace(/<w:p[ >\/]/g,'\n').replace(/<w:tab\/>/g,'\t').replace(/<[^>]+>/g,'').replace(/\n{3,}/g,'\n\n').trim();
  console.log(t);
})"
```

選單包含 5 份：
1. **事故分類**（3 項）
2. **災害類型表**（20 項）— 對應表單「事故類型」欄位
3. **災害原因分析 / 不安全行為**（15 項）
4. **不安全狀況**（9 項）
5. **基本原因**（9 項）

### Step 4：執行分析

#### Part A：公司內部要求欄位

依公司電子表單格式填寫，下拉選項從 Step 3 的選單中選擇：

1. **案件基本資訊**：從截圖謄寫所有欄位（含完整個資）
2. **事故經過**：從截圖完整謄寫
3. **改善對策（發生單位提出）**：從截圖完整謄寫
4. **災害原因分析**：從 15 項不安全行為中選擇最適合的選項 + 文字說明
5. **不安全狀況**：從 9 項中選擇最適合的選項 + 文字說明
6. **基本原因**：從 9 項中選擇最適合的選項 + 文字說明
7. **預防再發措施**：分立即/短期/長期三階段
8. **本案責任初步判定**：管理責任比例 vs 勞工責任比例 + 後續處理建議

#### Part B：進階分析（工安單位延伸）

運用專業分析方法進行深度分析：

1. **直接原因分析**：災害類型（從 20 項中選擇）、媒介物、接觸方式
2. **間接原因深度分析**：
   - 不安全行為分析：從 15 項中判定主因/次因/輔因
   - 不安全狀況分析：從 9 項中判定主因/次因/輔因
3. **基本原因深度分析**：從 9 項中判定主因/次因/輔因
4. **人為失效模式分析（HSG48）**：
   - 失誤類型：失手 Slip / 疏忽 Lapse / 判斷失誤 Mistake / 違規 Violation
   - 績效影響因素 PIF：職務 Job / 個人 Individual / 組織 Organisation
   - 參考：`職安/參考資料/職災分析方法/人為失效模式/HSG48_中文版.html`
5. **因果分析鏈（Why Tree）**：5-Why 逐層追問
   - 參考：事故調查指南附件 C
6. **改善對策建議**：依風險控制層級（消除→工程→管理→PPE）+ 期限
7. **發生單位改善對策評析**：評估方向正確性、補充建議、成效追蹤
8. **間接原因管理檢討**：制度/訓練/監督/文化四面向
9. **結論**：綜合摘要

### Step 5：產出 .docx 報告

使用報告模板腳本產出 Word 文件：

1. 準備 JSON 資料檔（包含所有分析內容）
2. 儲存 JSON 至：`職安/參考資料/公司職災資料/分析報告/{年份}/{檔名}_data.json`
3. 執行模板腳本產出 .docx：
```bash
node ~/.claude/skills/accident-analysis/report-template.js "{data.json路徑}" "{output.docx路徑}"
```

### Step 6：整理檔案

1. 報告存放：`職安/參考資料/公司職災資料/分析報告/{年份}/{YYYY-MM-DD}_{姓名}_{事故簡述}.docx`
2. 截圖備份：將使用者提供的截圖複製到 `職安/參考資料/公司職災資料/案件原始資料/{YYYY-MM-DD}_{姓名}/`
3. 回報完成，告知報告路徑

## 命名規則

- 報告檔名：`{YYYY-MM-DD}_{員工姓名}_{事故簡述}.docx`
- JSON 資料：`{YYYY-MM-DD}_{員工姓名}_{事故簡述}_data.json`
- 案件資料夾：`{YYYY-MM-DD}_{員工姓名}/`

## 注意事項

- 下拉選項只能從 `事故選單內容.docx` 中選擇，嚴禁自創
- 報告中完整記錄所有個資欄位（含身分證字號、出生年月日）
- 分析方法未來會新增，以 `職安/參考資料/職災分析方法/` 資料夾內容為準
- 產出報告前先與使用者確認截圖內容辨識是否正確
